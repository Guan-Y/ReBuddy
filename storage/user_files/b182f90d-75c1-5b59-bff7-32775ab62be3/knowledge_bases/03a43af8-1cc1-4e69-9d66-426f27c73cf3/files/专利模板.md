说明书
发明名称
【格式】不超过25个字，体现技术领域和核心创新点
示例：基于[信号特性]与[处理对象]的[技术效果][方法/装置]
技术领域
【格式】标准开场句 + 技术定位
示例：本发明涉及[大类技术领域]，尤其涉及一种[细分技术领域]的[具体应用方向]。
背景技术
【结构】
技术重要性说明：阐述本技术领域的基本原理、应用场景与行业价值
现有技术方案：综述当前主流方法的实现路径（分2-3类说明）
现有技术缺陷：指出至少2-3个具体技术问题，需与本发明创新点形成对应
撰写要点：
引用2-4篇代表性文献，标注[1][2]等
缺陷描述需具体化（如"未能显式利用xx先验特点"、"无法提供可解释性支持"）
避免贬低性词汇，使用"有待优化"、"存在提升空间"等客观表述
发明内容
【结构】三段论式撰写
1. 要解决的技术问题
【格式】针对背景技术缺陷，概括性提出本发明目的
示例：针对现有技术中[具体问题1]与[具体问题2]的不足，本发明提供一种[技术方案名称]，实现[技术效果1]与[技术效果2]的协同优化。
2. 技术方案
【格式】复制独立权利要求1，扩展为自然语言描述
要素：
明确方法/装置的组成部分
说明各步骤/模块的逻辑关系
使用"首先"、"接着"、"最终"等连接词增强可读性
3. 有益效果
【撰写模式】采用"特征→效果"的对应结构：
表格
复制
本发明技术特征	产生的技术效果	对比优势
[特征1：如主周期二维重构]	[效果1：显式分离周期内/间信息]	相比现有1D-CNN，表征能力提升xx%
[特征2：如双分支协同机制]	[效果2：噪声抑制与个性保留平衡]	相比简单拼接，鲁棒性提升xx%
[特征3：如可解释注意力权重]	[效果3：决策过程可追溯]	满足医疗AI可解释性监管要求
量化要求：至少提供1-2个可度量的性能指标（如准确率↑、延迟↓、鲁棒性↑）
附图说明
【格式】逐图说明，格式统一
示例：
图1为本发明提出的[方法名称]的流程图；
图2为本发明实施例提供的[模块名称]示意图；
图3为本发明实施例提供的[子模块]结构图；
...
图n为本发明[某一具体效果]的对比图。
具体实施方式
【结构】实施例 + 参数细节 + 公式推导
1. 实施例概述
【格式】本实施例提供一种[技术方案名称]，包括以下步骤：
2. 分步骤详细描述（每个步骤至少300字）
S1：数据预处理
子步骤分解：拆分为3-4个原子操作
参数具体值：滤波器类型（二阶巴特沃斯）、截止频率（0.5Hz）、数据集划分原则（跨患者独立验证）
数学公式：给出核心计算公式，标注变量含义
技术理由：解释为何选择该参数（如"为去除基线漂移对主周期提取的干扰"）
S2：核心特征提取
模块结构：说明子模块数量（如3个串联模块）、连接方式（残差连接）
递推公式：用LaTeX写出特征传递关系
维度变换：明确输入/输出张量形状（如R^(N×C) → R^(M×p)）
操作细节：分(1)(2)(3)点描述卷积核尺寸、滑动方向、激活函数选择
S3：创新机制实现
双分支并行处理：分别描述主/副分支的输入、处理逻辑、输出
注意力机制：写出权重计算公式（Softmax、缩放点积）
损失函数：若涉及对抗训练，写出损失函数形式（如L_adv = ||·||²）
融合策略：说明拼接方式（Concat）与维度（通道/特征维度）
S4：输出与可解释性
权重输出：强调α_i作为分类依据的同步输出
决策可视化：说明如何将权重映射到解剖区域
3. 实施例完整流程
【格式】用段落整合各步骤，形成连贯实现路径
作用：帮助审查员理解整体技术逻辑
